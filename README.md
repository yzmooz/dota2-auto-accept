# Dota 2 Auto Accept

Автоматическое принятие матча в Dota 2. Работает на **любом разрешении экрана**, не требует настройки — скачал, запустил, забыл.

---

## Быстрый старт (для новичков)

### Шаг 1 — Скачать программу

1. Перейдите в раздел **Releases** на странице проекта (справа на GitHub)
2. Скачайте файл `DotaAutoAccept.exe` (~30 МБ)
3. Сохраните его в удобное место, например на рабочий стол

> Антивирус может ругаться — это нормально для программ, скомпилированных через PyInstaller. Добавьте файл в исключения.

### Шаг 2 — Запустить

1. Дважды кликните по `DotaAutoAccept.exe`
2. Откроется окно программы с зелёной галочкой в заголовке

> Если хотите, чтобы программа запускалась автоматически при включении компьютера — перейдите во вкладку **«Настройки»** и включите **«Добавить в автозагрузку Windows»**.

### Шаг 3 — Настроить Telegram (необязательно)

Если хотите получать уведомления на телефон когда матч найден:

1. Откройте **Telegram** (на телефоне или компьютере)
2. Найдите бота: **@dota2_auto_accept_bot**
3. Напишите ему: `/start`
4. Бот ответит вам вашим **Chat ID** (набор цифр)
5. Скопируйте этот Chat ID
6. В программе перейдите во вкладку **«Telegram»**
7. Вставьте Chat ID в поле
8. Нажмите **«Проверить подключение»** — бот пришлёт тестовое сообщение
9. Включите переключатель **«Включить уведомления в Telegram»**

**Или ещё проще:** во вкладке «Telegram» нажмите кнопку **«🔍 Получить Chat ID от бота»**, затем напишите боту `/start` — Chat ID подставится автоматически.

### Шаг 4 — Использовать

1. Нажмите кнопку **▶ Старт** в программе
2. Запустите поиск матча в Dota 2 (кнопка «Играть» / «Найти матч»)
3. Когда матч найдётся — программа **автоматически** нажмёт кнопку «Принять»
4. Вам даже не нужно быть за компьютером!

## Как это работает

Программа **не сканирует экран постоянно**. Вместо этого она слушает системные события Windows и реагирует только когда:

- **Мигает иконка Dota 2** в панели задач (HSHELL_FLASH)
- **Окно Dota 2 выходит на передний план** (EVENT_SYSTEM_FOREGROUND)
- **Страховочный скан** — периодическая проверка каждые N секунд (настраивается)

Когда событие сработало, программа делает скриншот и ищет кнопку «Принять» одним или несколькими методами.

---

## Методы поиска кнопки

В программе используется **три метода обнаружения** кнопки «Принять». Каждый метод можно включать/выключать независимо — все активные работают одновременно, выбирается лучший результат:

### 1. Цветовой (HSV) 🟠

Находит кнопку **по зелёному цвету** через анализ HSV-маски скриншота.

- **Не зависит от разрешения** — работает на 1080p, 1440p, 4K, ультрашироких мониторах
- Не требует файла-шаблона
- Ищет прямоугольную зелёную область с правильным соотношением сторон в центральной части экрана
- Устойчив к масштабированию и изменению размера интерфейса

### 2. Шаблонный (Template Matching) 🔵

Сравнивает скриншот с **эталонным изображением** кнопки через `cv2.matchTemplate`.

- Шаблон сделан для 1920×1080 с русской локализацией
- Программа автоматически масштабирует шаблон (×0.85 – ×1.15)
- Точнее на «родном» разрешении, но может работать и на других

### 3. Клик в центр экрана 🔴

Просто кликает **в центр экрана** — там, где обычно находится кнопка «Принять».

- Самый простой и надёжный метод
- Используется как запасной вариант
- Настраиваемая «зона центра» — процент экрана, в пределах которого ожидается кнопка
- **Описание зоны центра**: если значение 15%, то на экране 1920×1080 это область ~288×162 пикселя ровно в центре. Чем больше значение, тем шире зона поиска (но выше шанс ложного срабатывания).

> **Рекомендация для большинства пользователей:** включите **Цветовой + Шаблонный** (по умолчанию). Если не работает — добавьте **Клик в центр**.

## FAQ

**Q: Работает ли на 1440p / 4K / ультрашироком мониторе?**  
A: Да! Цветовой метод (HSV) не зависит от разрешения. Шаблонный масштабирует картинку автоматически.

## Как это работает

Программа **не сканирует экран постоянно**. Вместо этого она слушает системные события Windows и реагирует только в момент, когда матч найден:

| Событие | Описание |
|---------|----------|
| **Мигание в панели задач** | Окно Dota 2 мигает красным — Windows посылает сигнал `HSHELL_FLASH` |

| **Страховочный скан** | Периодическая проверка экрана (каждые N секунд) на случай, если событие не пришло |

Когда одно из этих событий срабатывает, программа ищет зелёную кнопку «Принять» на экране и кликает по ней.

## Методы поиска кнопки

В программе используется **три метода обнаружения** кнопки «Принять». Каждый метод можно включать/выключать независимо — все активные работают одновременно, выбирается лучший результат:

### 1. Цветовой (HSV) 🟠

Находит кнопку **по зелёному цвету** через анализ HSV-маски скриншота.

- **Не зависит от разрешения** — работает на 1080p, 1440p, 4K, ультрашироких мониторах
- Не требует файла-шаблона
- Ищет прямоугольную зелёную область с правильным соотношением сторон в центральной части экрана
- Устойчив к масштабированию интерфейса Dota 2

**Как работает:**
1. Делается скриншот экрана
2. Переводится в цветовое пространство HSV
3. Применяется маска зелёного диапазона (H: 35–85, S: 80–255, V: 60–255)
4. Поиск ограничивается центральной 70% областью экрана (где находится попап матча)
5. Морфологическая очистка (close + open) убирает шум
6. Находятся контуры, фильтруются по размеру, соотношению сторон и заполненности
7. Выбирается лучший кандидат по уверенности

### 2. Шаблонный (Template Matching) 🔵

Классический **мультимасштабный `matchTemplate`** из OpenCV — сравнивает скриншот с эталонным изображением кнопки.

- Точнее при совпадении разрешения с эталоном
- Поставляемый шаблон снят на **1920×1080, русский интерфейс**
- Масштабируется автоматически (×0.85 — ×1.15) для других разрешений
- Для других языков/разрешений можно пересоздать шаблон через `make_template.py`

### 3. Клик в центр экрана 🔴

Простой клик **в центр экрана**, где обычно появляется кнопка «Принять».

- **Запасной вариант** — используется только если другие методы не нашли кнопку
- Размер зоны центра настраивается слайдером (5–40% экрана)
- Полезен на нестандартных разрешениях или при изменённом интерфейсе
- Гарантированное срабатывание — уверенность всегда 100%

# Dota 2 Auto Accept

Автоматически нажимает кнопку **«ПРИНЯТЬ»**, когда в Dota 2 находится матч.

Скрипт ждёт **настоящие события Windows**:

1. **`HSHELL_FLASH`** — окно Dota замигало в панели задач (то самое красное
   мигание при найденном матче) — главный триггер;
2. **`EVENT_SYSTEM_FOREGROUND`** — окно Dota вышло на передний план — запасной
   триггер.

Только в момент срабатывания триггера делается **один** поиск кнопки через
OpenCV (`matchTemplate` по эталону `images/accept_button.png`) и клик по ней.

## Установка

Нужен **Python 3.10+** и **Windows** (используются WinAPI-хуки).

```bash
git clone https://github.com/yzmooz/dota2-auto-accept.git
cd dota2-auto-accept


python -m venv .venv
source .venv/Scripts/activate      # PowerShell: .venv\Scripts\Activate.ps1

pip install -r requirements.txt
```

## Использование

Проверить, видит ли скрипт кнопку прямо сейчас (открой ready-check или покажи
скриншот кнопки на экране):

```bash
python accept_dota.py --test          # покажет уверенность, не кликая
python accept_dota.py --test --click  # то же + реальный клик (для проверки)
```

Рабочий режим — запусти перед поиском игры и оставь висеть:

```bash
python accept_dota.py
```

Остановка: **Ctrl+C** в консоли. (Увод курсора в левый верхний угол экрана —
это аварийная отмена клика pyautogui, а не выключение скрипта.)

## Свой эталон кнопки — `make_template.py`

Готовый эталон `images/accept_button.png` снят на **1920×1080, русский
интерфейс**. Поиск мультимасштабный и переживает небольшую разницу, но при
другом разрешении или языке интерфейса лучше пересоздать эталон под себя.
`make_template.py` сам находит зелёную кнопку «ПРИНЯТЬ» на твоём скриншоте и
вырезает её в `images/accept_button.png`.

```bash
# 1. Сделай скриншот всего экрана с видимой кнопкой
#    (Win+PrintScreen сохраняет в «Изображения/Снимки экрана»).
# 2. Сгенерируй эталон — зелёная кнопка находится автоматически:
python make_template.py "C:/путь/к/скриншоту.png"
```

Если автопоиск не сработал (кнопка другого цвета), задай область вручную в
пикселях `X Y W H`:

```bash
python make_template.py "C:/путь/к/скриншоту.png"  X Y W H
```

## Настройки

В начале файла `accept_dota.py`:

| Параметр           | Назначение                                                            |
|--------------------|-----------------------------------------------------------------------|
| `MATCH_THRESHOLD`  | Порог совпадения 0..1. Ложно кликает → повысь; не находит → понизь.   |
| `DEBOUNCE_SECONDS` | Пауза после клика, чтобы не нажимать повторно.                        |
| `SAFETY_SCAN_SEC`  | Фоновый страховочный скан раз в N сек (`0` = чисто событийный режим). |
| `RETRY_SECONDS`    | Сколько искать кнопку после триггера.                                 |

## Дисклеймер

Скрипт делает обычный клик по экрану и **не вмешивается в процесс игры** (не
читает и не меняет память Dota). Тем не менее используешь на свой риск.
